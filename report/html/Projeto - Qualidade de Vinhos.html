def slugify(c):
    # make strnig lower case
    c = c.lower()
    # remove unused white spaces
    c = c.strip()
    # replace multiple spaces with one
    c = re.sub('\s+', ' ', c)
    # replace spaces with underscore
    c = c.replace(" ", "_")
    return(c)

def chisq_of_df_cols(df, c1, c2):
    groupsizes = df.groupby([c1, c2]).size()
    ctsum = groupsizes.unstack(c1)
    # fillna(0) is necessary to remove any NAs which will cause exceptions
    return(chi2_contingency(ctsum.fillna(0)))

Ponto de atenção com a variável categórica¶

Ao utilizar implementaes computacionais dos algoritmos de classificao, temos que transformar as variveis categricas que esto em formato de string para um valor numrico. Porm, difcil encontrar um valor numrico que possa represent adequadamente essas variveis no dataset. Por exemplo, quando criamos a coluna type_encoded acima, atribumos o valor 1 para vinho tinto e 2 para vinho branco, mas isso no significa que esses nmeros tenham algum tipo de "representao de valor", ou seja, no temos como dizer que 1 est prximo de "vinho tinho". Para lidar com essa questo, vamos listar duas opes sugeridas na literatura cientfica:

  • 1-of-K Coding: consiste em transformar todas as variveis categricas em dummy vars. Esse mtodo, apesar de ter resultados satisfatrios, aumenta drasticamente o nmero de variveis presentes no dataset o que pode prejudicar a performance do algoritmo e tambm gerar overfitting. possvel ver uma descrio da tcnica em [5] (o artigo fala de clustering, mas a tcnica pode ser usada em situaes de aprendizado supervisionado tambm).
  • Two-step Method for Clustering Mixed numeric and categorical data (TMCM) [6]: o algoritmo foi desenvolvido para conseguir lidar com variveis categricas e numricas simultaneamente na clusterizao. Ele dividido em duas etapas principais: a primeira a manipulao de variveis categricas de acordo com a teoria de co-ocorrncia, que analisa o relacionamento entre as variveis categricas e gera um valor numrico razovel (e no binrio) para elas; a segunda consiste na aplicao do K-means em duas etapas. Nesse estudo, foi feita a implementao da primeira etapa do algoritmo e o cdigo pode ser encontrado no arquivo tmcm_feature_engineering.py.

Implementação do modelo¶

Utilizaremos o modelo de rvores de deciso para resolver este desafio. Ele um modelo no paramtrico de classificao e que aprende algumas regras de deciso a partir dos dados inputados no mesmo. Entre suas vantagens esto:

  • Facilidade de interpretao ( possvel explicar cenrios usando lgica booleana)
  • Capaz de lidar com um problema de multi classes como o deste desafio

Um grande trade-off desse modelo a possibilidade de criar rvores demasiadamente complexas e que podem criar overfitting. Para evitar que isso acontea, vamos desenhar as curvas de aprendizado de diferentes rvores treinadas e variando a altura da mesma (max_depth).

Ademais, usaremos information gain como mtrica de custo na criao da rvore. Basicamente ela usa o decaimento da entropia (medida de homogeneidade de um conjunto de dados) para selecionar a melhor quebra dos dados em uma feature. Como mtrica de avaliao da qualidade do modelo, usaremos como mtrica principal a acurcia, que nos indicar o percentual dos exemplos que classificamos corretamente, e tambm usaremos as mtricas de preciso e recall como secundrias a fim de identificar potenciais vises do modelo.

Seleção de features¶

Vamos listar abaixo algumas formas de selecionar as melhores features para o modelo:

  • Conhecimento de contexto: consiste em usar o conhecimento de negcio existente para remover features que no sejam relevantes ou que no possam ser usadas por questes como regulamentao ou estratgia de negcio.
  • Mtodos de filtragem: so usados geralmente como etapas de pr processamento dos dados e no dependem de algoritmos de aprendizado de mquina especficos. As features so selecionadas a partir de testes estatsticos relevantes.
  • Mtodos wrapper: consiste em usar um subconjunto de features para treinar e testar o modelo e, dependendo dos resultados deste modelo, decidido pela adio ou remoo das features. Basicamente se trata de um problema de busca.

Neste problema, usaremos os testes estatsticos das sees anteriores para decidir que features remover. A partir do que foi exposto na seo de anlis exploratria, vimos que a varivel residual_sugar tem um correlao muito baixa com qualidade (prxima de zero), logo no a usaremos em nosso modelo.

Conclusões¶

Neste projeto foi implementado um modelo de rvore de deciso para predizer a qualidade de vinhos. A qualidade dos vinhos apresentou uma distribuio normal e tambm continha pouqussimos exemplos com qualidade muito boa ou muito ruim. Para lidar com isso, optou-se por tornar o problema numa classificao binria.

Tambm usamos uma tcnica baseada na teoria de co-ocorrncia para criar valores numricos apropriados para features categricas do dataset. Implementamos um modelo de rvore de deciso que apresentou acurcia de cerca de 75%. Por fim, exploramos algumas variantes da soluo proposta para averiguar se as mtricas dos modelos melhorariam.

Para trabalhos futuros, vislumbram-se as seguintes possibilidades:

  • Explorar outros algoritmos de classificao
  • Aplicar a tcnica de 1-of-K Coding para lidar com a varivel categrica type
  • Utilizar a tcnica de oversampling para tratar melhor os dados
  • Tentar adquirir mais dados amostras para o dataset, principalmente de vinhos muito bons e muito ruins para que, a partir deles seja possvel desenvolver um classificador multi-classe confivel.
